# Production checklist

Не открывайте регистрацию для посторонних пользователей, пока каждый обязательный
пункт ниже не проверен на том же домене и сервере, которые будут использоваться в
production.

## Инфраструктура

- [ ] DNS направлен на сервер, Nginx принимает только нужный домен, HTTPS-сертификат обновляется автоматически.
- [ ] `YT_LOADER_SECURE_COOKIES=true`, `YT_LOADER_ALLOWED_HOSTS` содержит production-домен, API docs выключены.
- [ ] PostgreSQL и Uvicorn доступны только во внутренней Docker-сети/на `127.0.0.1`.
- [ ] В `.env` права `600`; SMTP, PostgreSQL и ЮKassa используют отдельные production-секреты.
- [ ] Настроены алерты на HTTP 503 `/api/health`, перезапуски контейнера, заполнение диска и ошибки SMTP/worker.
- [ ] Проверена пропускная способность CPU, диска и сети на типичном видео и максимальном пакете.

## Аккаунты и почта

- [ ] `YT_LOADER_REQUIRE_EMAIL_VERIFICATION=true`; письмо подтверждения приходит и не попадает в спам.
- [ ] Одноразовая ссылка подтверждает аккаунт, повторное использование отклоняется.
- [ ] Сброс пароля приходит только по HTTPS, меняет пароль и отзывает ранее открытые сессии.
- [ ] Администратор создан через `manage_users.py create-admin`, публичная регистрация включена осознанно.

## Деньги

- [ ] Тестовый, затем боевой `shopId` и ключ ЮKassa установлены без сохранения в Git.
- [ ] Webhook подписан на `payment.succeeded` и `payment.canceled`; IP-фильтр не отключён.
- [ ] Проверены успешная/отменённая оплата, дубликат webhook, задержанный webhook и автопродление.
- [ ] `audit-payments` и `audit-credits` завершаются без `MISMATCH`.
- [ ] Бухгалтер подтвердил способ формирования чеков, ставки НДС и настройки 54-ФЗ.
- [ ] Определены и протестированы правила возврата, отмены и спорного платежа.

## Данные и восстановление

- [ ] `backup-postgres.sh` запускается ежедневно, копия уходит на отдельное физическое хранилище.
- [ ] На отдельном тестовом PostgreSQL выполнено восстановление через `restore-postgres.sh`.
- [ ] После восстановления пройдены финансовые сверки и выборочная загрузка пользовательского задания.
- [ ] Сроки хранения видео, webhook и логов соответствуют опубликованной политике.

## Правовые и продуктовые условия

- [ ] Опубликованы реквизиты продавца, оферта, политика конфиденциальности, согласие на обработку данных и контакты поддержки.
- [ ] Пользователь до оплаты видит цену, период подписки, автопродление и способ его отключения.
- [ ] Пользователь принимает правила использования и подтверждает права на обрабатываемые видео.
- [ ] Проверены требования YouTube, платёжного провайдера и применимого законодательства.
- [ ] Служба поддержки знает, как найти платёж, отменить продление, оформить возврат и сохранить финансовый аудит.

## Release gate

```bash
alembic current
alembic check
docker compose config --quiet
docker compose ps
curl -fsS https://YOUR_DOMAIN/api/health
docker compose exec yt-loader python manage_users.py audit-payments
docker compose exec yt-loader python manage_users.py audit-credits
```

Ожидаемая ревизия Alembic: `5dbe4a9120ef`. Health должен вернуть `status`, `database`
и `workers` со значением `ok`.

Для самого первого перехода со старой JSON/Basic Auth версии используй
`deploy/first-saas-upgrade.sh`; обычный `docker compose up -d --build` не создаёт
предварительную копию legacy-данных и не выполняет поэтапное переключение.
